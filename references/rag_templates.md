# 初步故障问答（RAG）答案框架与模板

本文件为 OpenClaw Agent 提供了在处理火山引擎常见故障问题时的回答框架和模板。Agent 在生成答案时，应遵循“问题重述 → 原因线索 → 建议动作 → 风险提示 → 如需人工”的结构，并结合从云资源巡检、监控数据中获取的实际信息进行填充。

## 一、 标准答案框架

当用户提出故障类问题时，无论场景如何，都应遵循以下五步结构组织回答：

---

**1. 问题重述 (Restate the Problem)**
> **目的**：确认 Agent 正确理解了用户的问题。
> **话术**：“您好，我理解您的问题是关于 [**您遇到的具体问题，例如：部署在火山引擎上的网站突然无法访问**]。”

**2. 原因线索 (Provide Potential Clues)**
> **目的**：基于已有的数据（如巡检报告、监控指标）给出最可能的原因线索，而不是直接下定论。
> **话术**：“根据我刚刚做的快速检查，发现了以下几条可能相关的线索：”
> - **线索一**：[例如：云服务器 `i-xxxx` 的 CPU 使用率在过去 10 分钟内达到了 100%。]
> - **线索二**：[例如：安全组 `sg-yyyy` 的入方向规则没有放开 80 或 443 端口。]
> - **线索三**：[例如：负载均衡 `clb-zzzz` 的后端服务器池中，有 1 台实例健康检查失败。]
> **兜底话术**（如果没有明显线索）：“目前从资源层面看，各项核心指标（如 CPU、内存、网络）暂未发现明显异常。”

**3. 建议动作 (Suggest Actions)**
> **目的**：提供清晰、可执行的下一步操作建议，最好是用户可以通过 Agent 直接发起的操作。
> **话术**：“基于以上线索，我建议您可以尝试以下操作来进一步排查或解决问题：”
> - **动作一**：[例如：**重启实例** `i-xxxx`，这可能会临时缓解因进程卡死导致的高负载问题。您需要执行此操作吗？]
> - **动作二**：[例如：为您**调整安全组** `sg-yyyy`，添加入方向规则以允许公网访问 80 端口。是否需要我为您生成 `dry-run` 预览？]
> - **动作三**：[例如：**登录实例**排查应用日志，检查是否有应用层面的错误。]

**4. 风险提示 (State the Risks)**
> **目的**：对建议的变更类操作进行明确的风险提示，建立用户信任。
> **话术**：“请注意，执行以上操作可能会有以下影响：”
> - **重启实例**：会导致服务短暂中断，中断时长取决于您的应用启动速度。
> - **修改安全组**：向公网开放端口会增加服务器被攻击的风险，请确保您的应用已做好安全加固。
> - **费用相关**：开机/创建资源会产生费用，请关注您的账户预算。

**5. 如需人工 (Escalate to Human)**
> **目的**：当自动排查无法解决问题时，提供清晰的求助路径。
> **话术**：“如果以上步骤未能解决您的问题，或者您不确定如何操作，我可以为您**生成一份工单**，包含了所有诊断信息，以便技术支持专家快速介入。需要吗？”

## 二、 常见场景模板

### 场景一：网站不可用

**用户问题**：“为什么我的网站突然打不开了？” / “应用访问超时。”

**填充示例**：
> **问题重述**：您好，我理解您的问题是关于部署在火山引擎上的网站（或应用）突然无法访问。
>
> **原因线索**：根据快速巡检，我注意到：
> - 云服务器 `i-web-server-01` 的网络出带宽在过去 5 分钟内持续为 0。
> - 该实例的状态（Status）显示为 `Stopped`。
>
> **建议动作**：
> - 我可以为您**启动实例** `i-web-server-01`。这通常可以解决因实例意外关机导致的问题。您需要我执行 `dry-run` 预览吗？
>
> **风险提示**：
> - 启动实例后，将开始按小时或按需计费。请确保您的账户余额充足。
>
> **如需人工**：
> - 如果启动后问题依旧，可能涉及更复杂的网络或应用配置问题。届时我可以为您创建工单。

### 场景二：费用异常

**用户问题**：“为什么这个月账单突然变高了？” / “我的云费用异常。”

**填充示例**：
> **问题重述**：您好，我理解您对本月火山引擎的费用增长有疑问。
>
> **原因线索**：通过分析近期的资源和用量（基于 `ark_usage_cost.py` 的输出），我发现：
> - 项目 `data-processing` 下，方舟 Ark 模型的 `total_tokens` 调用量从日均 100 万突增至昨天的 2000 万。
> - 您账户下有 5 个 `ecs.g2ie.2xlarge` 类型的实例处于 `Running` 状态，但它们过去 7 天的平均 CPU 利用率低于 1%。
>
> **建议动作**：
> - **详细分析 Ark 用量**：我可以按调用方（caller）或模型（model）聚合，找出导致 token 量剧增的具体源头。需要执行吗？
> - **优化低利用率实例**：对于那 5 台低负载实例，建议您**关机**以停止计费，或**降配**到更经济的实例类型。我可以列出这些实例的 ID 供您决策。
>
> **风险提示**：
> - 关机或降配可能会影响相关业务，请在操作前确认这些实例的用途。
>
> **如需人工**：
> - 如果您对计费明细仍有疑问，我可以为您生成一份包含用量明细和异常点的工单，提交给商务或技术支持团队。

### 场景三：实例无法 SSH

**用户问题**：“我的服务器连不上了。” / “ECS 实例无法 SSH。”

**填充示例**：
> **问题重述**：您好，我理解您的问题是无法通过 SSH 连接到您的火山引擎 ECS 实例。
>
> **原因线索**：我检查了实例 `i-dev-test-01` 的配置和状态，发现：
> - 该实例的安全组 `sg-internal-only` 的入方向规则中，未包含允许从您的公网 IP 访问 22 端口的策略。
> - 实例当前正在运行，且 CPU 和内存使用率正常。
>
> **建议动作**：
> - 我可以为您**修改安全组** `sg-internal-only`，临时添加一条规则，允许从 IP `XXX.XXX.XXX.XXX`（您的当前出口 IP）访问 22 端口。需要为您生成 `dry-run` 预览吗？
>
> **风险提示**：
> - 向公网暴露 SSH 端口存在安全风险。建议您仅临时开放，或使用 VPN 等更安全的方式接入。
>
> **如需人工**：
> - 如果添加安全组规则后仍然无法连接，可能涉及实例内部的 SSH 服务配置或操作系统防火墙问题。您可能需要通过控制台的 VNC 登录进行排查，或联系技术支持。
